## US-01: 画像の一覧・プレビュー・メタデータ確認

**ロール:** ターミナルを使わないコンテンツ作成者
**目的:** サイト全体にアップロードされた画像（`src/image/` 配下）をエディタUIから確認したい
**価値:** どんな画像が存在するかをブラウザだけで把握でき、アップロードが正しく完了したかも確認できる

### シナリオ 1: 画像一覧を開くと全画像が表示される

```
Given サイトに画像がアップロードされている
When  エディタUIで画像ライブラリを開く
Then  src/image/ 配下の全画像がサムネイル一覧として表示される
```

### シナリオ 2: 各画像のメタデータが確認できる

```
Given 画像ライブラリに画像が一覧表示されている
When  画像を選択・フォーカスする
Then  その画像のファイル名・ファイルサイズ・解像度が表示される
And   エディタ経由で追加された画像であれば追加日時も表示される
```

### シナリオ 3: エディタ導入以前からある画像は追加日時なしで表示される

```
Given エディタを経由せずに配置された画像が src/image/ 配下に存在する
When  その画像を選択・フォーカスする
Then  ファイル名・ファイルサイズ・解像度は表示される
And   追加日時は「不明」等、記録がないことが分かる形で表示される
```

### シナリオ 4: 画像が1枚もない場合

```
Given サイトに画像が1枚もアップロードされていない
When  エディタUIで画像ライブラリを開く
Then  「画像がありません」などの空状態が表示される
```

## US-02: 画像の削除

**ロール:** ターミナルを使わないコンテンツ作成者
**目的:** 不要な画像をエディタUIから削除したい
**価値:** ターミナルを一切使わずに不要な画像を片付けられ、自分の知らないうちにMarkdownが書き換わることも、公開中のサイトをうっかり壊すこともない

### シナリオ 1: どの記事からも参照されていない画像を削除できる

```
Given 画像ライブラリに画像が表示されている
And   その画像はいずれの記事からも参照されていない
When  削除操作を行う
Then  画像ファイルが削除される
And   画像ライブラリの一覧からその画像が消える
```

### シナリオ 2: 参照されている画像の削除は、参照記事の提示と選択を経る

```
Given 画像ライブラリに画像が表示されている
And   1つ以上の記事がその画像を参照している
When  削除操作を行う
Then  削除は即座には実行されない
And   その画像を参照している記事の一覧が提示される
And   一覧のうち公開済みの記事が識別できる
And   「参照はそのままにして削除」「参照も除去して削除」「中止」に相当する選択肢が提示される
      （※選択肢の文言は設計フェーズで検討する）
```

### シナリオ 3: 「参照はそのままにして削除」を選ぶ

```
Given 参照されている画像の削除で、参照記事一覧と選択肢が提示されている
When  参照はそのままにして削除する選択を行う
Then  画像ファイルが削除される
And   参照していた記事は変更されない
And   画像ライブラリの一覧からその画像が消える
```

### シナリオ 4: 「参照も除去して削除」を選ぶ

```
Given 参照されている画像の削除で、参照記事一覧と選択肢が提示されている
When  参照も除去して削除する選択を行う
Then  画像ファイルが削除される
And   参照していた記事から該当の画像参照が除去される
And   画像ライブラリの一覧からその画像が消える
```

### シナリオ 5: 「中止」を選ぶ

```
Given 参照されている画像の削除で、参照記事一覧と選択肢が提示されている
When  中止する選択を行う
Then  画像ファイルは削除されない
And   記事も変更されない
```

## US-03: 画像の移動（パスの付け替え）

**ロール:** ターミナルを使わないコンテンツ作成者
**目的:** 画像のパス（置き場所とファイル名）をエディタUIから付け替えたい。ファイル名だけの変更（改名）は、同じ置き場所での付け替えという特殊な場合
**価値:** ターミナルを一切使わずに画像の置き場所とファイル名を整理でき、参照の扱いも自分で選べる。追加時に配置を慎重に決めなくても、後から付け替えられる

### シナリオ 1: どの記事からも参照されていない画像を移動できる

```
Given 画像ライブラリに画像が表示されている
And   その画像はいずれの記事からも参照されていない
When  新しいパス（別の置き場所）を入力して移動操作を行う
Then  画像ファイルが新しいパスに移る
And   画像ライブラリの一覧で新しいパスに表示される
```

### シナリオ 2: ファイル名だけの変更（改名）もできる

```
Given 画像ライブラリに画像が表示されている
And   その画像はいずれの記事からも参照されていない
When  置き場所は変えずファイル名だけを変えて移動操作を行う
Then  画像ファイルが新しい名前に変更される
And   画像ライブラリの一覧で新しいファイル名が表示される
```

### シナリオ 3: 参照されている画像の移動は、参照記事の提示と選択を経る

```
Given 画像ライブラリに画像が表示されている
And   1つ以上の記事がその画像を参照している
When  新しいパスを入力して移動操作を行う
Then  移動は即座には実行されない
And   その画像を参照している記事の一覧が提示される
And   一覧のうち公開済みの記事が識別できる
And   「参照も新しいパスに書き換えて移動」「参照はそのままにして移動」「中止」に相当する選択肢が提示される
      （※選択肢の文言は設計フェーズで検討する）
```

### シナリオ 4: 「参照も書き換えて移動」を選ぶ

```
Given 参照されている画像の移動で、参照記事一覧と選択肢が提示されている
When  参照も書き換えて移動する選択を行う
Then  画像ファイルが新しいパスに移る
And   参照していた記事の画像参照が新しいパスに書き換えられる
And   画像ライブラリの一覧で新しいパスに表示される
```

### シナリオ 5: 「参照はそのままにして移動」を選ぶ

```
Given 参照されている画像の移動で、参照記事一覧と選択肢が提示されている
When  参照はそのままにして移動する選択を行う
Then  画像ファイルが新しいパスに移る
And   参照していた記事は変更されない
And   画像ライブラリの一覧で新しいパスに表示される
```

### シナリオ 6: 「中止」を選ぶ

```
Given 参照されている画像の移動で、参照記事一覧と選択肢が提示されている
When  中止する選択を行う
Then  画像ファイルのパスは変更されない
And   記事も変更されない
```

### シナリオ 7: 移動先にすでに同じパスの画像が存在する場合

```
Given 画像ライブラリに画像が表示されている
When  既存の別の画像と同じパスになる入力で移動操作を行う
Then  移動は実行されない
And   パスが重複している旨がユーザーに伝わる
```

### シナリオ 8: 画像がUIから管理できなくなる付け替えは受け付けられない

```
Given 画像ライブラリに画像が表示されている
When  画像でない拡張子（例: .md）に変わる入力で移動操作を行う
Then  移動は実行されない
And   受け付けられない旨がユーザーに伝わる
And   画像は引き続き画像ライブラリから管理できる
```

## US-04: 画像ライブラリからの画像追加

**ロール:** ターミナルを使わないコンテンツ作成者
**目的:** 記事編集を経由せずに、画像をサイトに追加したい
**価値:** 記事にまだ載せない画像も、ブラウザだけでサイトに取り込んでおける

※ 追加の導線は、記事の新規作成と同じ「新規作成」タブ（新しい資源の作成として記事と画像を同じ導線で扱う。設計の経緯は test-tree.md の US-04 フェーズ参照）

### シナリオ 1: 新規作成から画像を追加できる

```
Given エディタUIを開いている
When  新規作成タブで画像ファイルを選び、配置パスを指定して追加する操作を行う
Then  画像が src/image/ 配下に保存される
And   画像ライブラリの一覧にその画像が現れる
And   追加日時が記録される
```

### シナリオ 2: 追加しただけでは公開されない

```
Given 画像ライブラリから画像を追加した
And   その画像はどの記事からも参照されていない
Then  その画像はリモートに公開されていない
```

## US-05: 参照に連動した公開状態

**ロール:** ターミナルを使わないコンテンツ作成者
**目的:** 画像の公開・非公開を個別に操作せず、記事からの参照の有無で自動的に決まってほしい
**価値:** 画像の公開タイミングを意識する必要がなく、使われなくなった画像がリモートに残り続けることもない

### シナリオ 1: 参照する記事の公開と共に画像が公開される

```
Given 未公開の画像がある
And   その画像を参照する記事がある
When  その記事を公開する
Then  画像も記事と共に公開される
```

### シナリオ 2: 公開済みのすべての記事から参照が失われると、同期のタイミングで取り除かれる

```
Given 公開済みの画像がある
And   その画像を参照する公開済みの記事が1つだけある
And   ローカルでその記事から参照が除去されている
When  その記事の更新を公開する
Then  記事の更新がリモートに反映される
And   画像がリモートから取り除かれる
```

### シナリオ 3: 他の公開済み記事がまだ参照していれば取り除かれない

```
Given 公開済みの画像がある
And   その画像を参照する公開済みの記事が2つある
And   ローカルで一方の記事から参照が除去されている
When  その記事の更新を公開する
Then  記事の更新がリモートに反映される
And   画像はリモートに残る
```

### シナリオ 4: ローカルで参照を消しただけでは取り除かれない

```
Given 公開済みの画像がある
And   その画像を参照する公開済みの記事がある
And   ローカルでその記事から参照が除去されている（未公開）
When  別の記事の公開・更新など、他の同期操作を行う
Then  画像はリモートに残る（リモートの記事がまだ参照しているため）
```

## US-06: 検出できない参照の宣言

**ロール:** サイトを管理するコンテンツ作成者
**目的:** テンプレート等、エディタが自動検出できない場所で使っている画像を宣言して守りたい
**価値:** 記事から参照されていない画像でも、宣言しておけば参照切れ扱いや自動非公開でサイトが壊れることがない

### シナリオ 1: 宣言を付与できる

```
Given 画像ライブラリに画像が表示されている
When  その画像に「エディタの検出外で使われている」ことを宣言する
Then  宣言が保存され、画像ライブラリ上で宣言があることが分かる
```

### シナリオ 2: 宣言された画像は自動非公開の対象にならない

```
Given 宣言が付与された公開済みの画像がある
And   その画像はどの公開済み記事からも参照されていない
When  同期操作を行う
Then  画像はリモートに残る
```

### シナリオ 3: 宣言を解除すると通常の導出に従う

```
Given 宣言が付与された画像がある
When  宣言を解除する
Then  以後、その画像の公開状態は記事からの参照の有無に従って決まる
```

## US-07: 画像も記事と同じナビゲーションで扱える

**ロール:** ターミナルを使わないコンテンツ作成者
**目的:** 画像の選択・表示を、記事ファイルと同じナビゲーションの理解（URLが資源を特定し、表示はURLから再構成される）で扱いたい
**価値:** ファイルと画像で操作の意味が変わらず、リロード・戻る・URLの共有といったブラウザの当たり前の操作が画像にも通用する

### シナリオ 1: リロードしても同じ表示が再現される

```
Given 画像ライブラリで画像を選択し、その詳細が表示されている
When  ブラウザをリロードする
Then  同じ画像の詳細表示が再現される
```

### シナリオ 2: URLの直打ちで画像を開ける

```
Given ある画像を特定するURLがある
When  そのURLをブラウザで直接開く
Then  その画像の詳細が表示される
```

### シナリオ 3: サイドバーで選択中の画像が識別できる

```
Given ある画像の詳細が表示されている
When  サイドバーの画像一覧を見る
Then  表示中の画像が選択状態として識別できる（ファイル一覧のアクティブ表示と同様）
```

### シナリオ 4: ブラウザの「戻る」で遷移前の表示に戻れる

```
Given 記事を開いた状態から画像を選択し、画像の詳細が表示されている
When  ブラウザの「戻る」を操作する
Then  遷移前の記事の表示に戻る
```
