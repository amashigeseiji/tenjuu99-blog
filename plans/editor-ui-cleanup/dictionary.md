## エディタ

### 新規作成

en: NewFileCreation

**関係:**
- `references`: [フロントマターテンプレート](../../../docs/dictionary.md#フロントマターテンプレート) — 作成時にテンプレートを選択できる
- `references`: [公開ステータス](../../../docs/dictionary.md#公開ステータス) — 作成直後は「未公開」ステート
- `references`: [保存](../../../docs/dictionary.md#保存) — 作成と同時にローカル保存が行われる
**src:** （ウォークスルーで記入）
**発見:** problem.md v1 申し送り
**状態:** 新規

### 自動保存

en: AutoSave
**定義:** テキストエリアへの入力を契機に、saveボタンの操作なしにファイルがローカルに[保存](#保存)（docs/dictionary.md#保存）されること。[リアルタイムプレビュー更新](#リアルタイムプレビュー更新)（docs/dictionary.md#リアルタイムプレビュー更新）と同様に「書く」行為の流れを中断しない。saveボタンはこの機能の導入に伴い削除される。
**関係:**
- `references`: [保存](../../../docs/dictionary.md#保存) — 保存の自動化された形態
- `references`: [リアルタイムプレビュー更新](../../../docs/dictionary.md#リアルタイムプレビュー更新) — 同じトリガー（テキスト入力の停止）で発火する
- `references`: [テキスト入力](../../../docs/dictionary.md#テキスト入力) — テキスト入力の停止がトリガーとなる
**src:** （ウォークスルーで記入）

### 自動保存初期化器

en: AutoSaveInitializer
**定義:** テキストエリアへの入力イベントを受け取り、[デバウンサー](../../../docs/dictionary.md#デバウンサー)を介して入力停止を検知し、保存コールバックを呼び出す装置。[プレビュー自動更新器](../../../docs/dictionary.md#プレビュー自動更新器)と同構造だが、保存を担う。
**関係:**
- `references`: [自動保存](#自動保存) — 自動保存の JS 側実現

### 保存エンドポイント

en: SaveEndpoint
**定義:** ファイル名と内容を受け取り、ローカルファイルシステムに書き込む POST エンドポイント（`/save`）。保存後はリダイレクトせず `{success: true}` を返す。[自動保存](#自動保存)と[新規作成](#新規作成)の両方から利用される共有ノード。`server/editor.js` を置き換える。
**関係:**
- `references`: [自動保存](#自動保存) — 自動保存の保存先
- `references`: [新規作成](#新規作成) — 新規ファイル作成時の保存先
**src:** `packages/editor/server/save.js`（ウォークスルーで記入）

---

### 新規作成UI

en: NewFileCreationUI
**定義:** 「新規作成」ボタンとダイアログで構成されるブラウザUI。ファイル名の入力と[フロントマターテンプレート](../../../docs/dictionary.md#フロントマターテンプレート)の表示・選択を担う。確定時に[保存エンドポイント](#保存エンドポイント)を呼び出してファイルを作成し、エディタで開く。
**関係:**
- `references`: [新規作成](#新規作成) — 新規作成操作を UI として具現化する
- `references`: [フロントマターテンプレート](../../../docs/dictionary.md#フロントマターテンプレート) — ディレクトリに応じたテンプレートを表示する
- `references`: [保存エンドポイント](#保存エンドポイント) — 確定時に呼び出す
**src:** `packages/editor/js/editor.js`（ウォークスルーで記入）

**再定義（docs/dictionary.md との差分）**

### 保存（再定義）

**再定義元:** docs/dictionary.md#保存
**現在の定義:** テキストエリアの内容をローカルファイルシステムに書き込む操作。エディタの save ボタンで実行され、成功すると対応するページへリダイレクトされる。
**新しい定義:** テキストエリアの内容をローカルファイルシステムに書き込む操作。[自動保存](#自動保存)によって実行されるか、手動操作によって実行される。save ボタンが削除されるため、通常は自動保存が担う。
**変更理由:** 自動保存の導入により、save ボタンへの依存が解消される。保存の概念自体は変わらないが、「save ボタンで実行される」という手段記述を除去する。
**影響範囲（再定義の場合）:** docs/dictionary.md#保存 の定義文。実装の挙動変更は editor.js の submit / save フロー。
**状態:** 再定義（既存「保存」の意味を変更）
