## US-01: 新規ファイルの作成

**ロール:** コンテンツ作成者（非技術者）
**目的:** 新しいページを独立した操作で作る
**価値:** 新規作成がファイル選択と混在せず、迷わずに操作できる

### シナリオ 1: 新規作成ボタンからファイルを作る

```
Given エディタ画面が開いている
When 「新規作成」ボタンを押す
Then ファイル名を入力できる状態になる
```

### シナリオ 2: ファイル名とテンプレートを指定して作成する

```
Given 「新規作成」操作でファイル名入力の状態になっている
  And そのディレクトリにはテンプレートが設定されている
When ファイル名を入力し、テンプレートを選択して確定する
Then 指定したテンプレートの内容が挿入された状態でエディタに開かれる
  And ファイルはローカルに保存されている
  And ファイルは公開されていない（公開ステータスが「未公開」）
```

### シナリオ 3: テンプレートが設定されていないディレクトリで作成する

```
Given 「新規作成」操作でファイル名入力の状態になっている
  And そのディレクトリにはテンプレートが設定されていない
When ファイル名を入力して確定する
Then 空の状態でエディタに開かれる
  And ファイルはローカルに保存されている
  And ファイルは公開されていない（公開ステータスが「未公開」）
```

## US-02: 書いた内容の自動保存

**ロール:** コンテンツ作成者
**目的:** save ボタンを押さずに書いた内容を失わない
**価値:** 保存するタイミングを気にせず書くことに集中できる

### シナリオ 1: 書いた内容が自動的に保存される

```
Given 既存のファイルがエディタに開かれている
When テキストを書く
Then 明示的に save ボタンを押さなくても、書いた内容がローカルに保存されている
```

### シナリオ 2: 新規作成したファイルに書いた内容も保存される

```
Given 新規作成したファイルがエディタに開かれている
When テキストを書く
Then 書いた内容がローカルに保存されている
```

## US-03: プレビューを確認しながら書く

**ロール:** コンテンツ作成者
**目的:** 書いた内容の見た目を確認しながら書く
**価値:** preview ボタンを押す操作なしに、書いた内容がどう見えるかを常に確認できる

### シナリオ 1: テキストを書くとプレビューが更新される

```
Given ファイルがエディタに開かれている
  And preview ボタンは存在しない
When テキストを書く
Then プレビューペインに最新の内容が反映される
```

---

## US-04: ファイルのナビゲーション

**ロール:** コンテンツ作成者
**目的:** 編集したいファイルをサイドバーから選んで開く
**価値:** ファイル選択の手段がサイドバーに一元化されており、どこで選べばよいか迷わない

### シナリオ 1: サイドバーのファイルをクリックして開く

```
Given エディタ画面のサイドバーにディレクトリツリーが表示されている
  And ファイル選択セレクトボックスは存在しない
When サイドバーのファイルをクリックする
Then そのファイルがエディタに開かれる
```
