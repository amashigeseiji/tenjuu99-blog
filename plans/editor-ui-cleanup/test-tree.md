# テストツリー: editor-ui-cleanup

**作成:** 2026-06-27
**更新:** 2026-06-28（findings F-01〜F-04, F-06, F-07 対応）

## できるのツリー

```
エディタは整合した操作フローを提供できる [editor]
├── 自動保存器は入力を契機に保存できる [editor]  ✅
│   ├── 自動保存初期化器は入力停止を検知して保存コールバックを呼べる [editor]  ✅
│   └── 保存エンドポイントはリダイレクトなしにファイルを保存できる [editor]  ✅  ← 共有ノード
└── 新規作成器はファイルを新規作成できる [editor]
    ├── 新規作成UIはファイル名とテンプレートを受け付けられる [editor]  ← 手動確認のみ（ブラウザUI）
    │   ├── Enterキーで作成を確定できる [editor]  ← NEW（手動確認のみ）
    │   ├── テンプレートをセレクトボックスで選択できる [editor]  ← NEW（手動確認のみ）
    │   ├── 既存ファイル名のとき重複エラーを表示できる [editor]  ← NEW（手動確認のみ）
    │   └── 作成後にサイドバーを更新できる [editor]  ← NEW（手動確認のみ）
    ├── 保存エンドポイントはリダイレクトなしにファイルを保存できる [editor]  ✅  ← 共有ノード
    └── 新規作成エンドポイントは既存ファイルへの上書きを拒否できる [editor]  ← NEW（ユニットテスト可）
```

## モジュール境界

| ノード | 実装 | テスト可能性 |
|--------|------|-------------|
| 自動保存初期化器 | `packages/editor/js/autoSaveInitializer.js` | Node.js テスト可 |
| 保存エンドポイント | `packages/editor/server/save.js`（`/save` POST） | Node.js テスト可 |
| 新規作成エンドポイント | `packages/editor/server/save.js`（`saveFile` に `createOnly` オプション追加） | Node.js テスト可 |
| 新規作成UI | `packages/editor/js/editor.js` 内ダイアログ | 手動確認のみ |
| サイドバー取得 | `packages/editor/server/get_sidebar.js`（新設） | 手動確認のみ |
| HTML変更 | `packages/editor/template/editor.html` | ユーザーストーリーテストで確認 |

## 利用仮説

- 使ったらこうなるはず: 「新規作成」ボタンを押してファイル名を入力し確定すると、テンプレート付きでエディタに開かれる
- 使ったらこうなるはず: Enter キーを押しても「作成」ボタンと同じ動作になる
- 使ったらこうなるはず: テンプレートをセレクトボックスで選べる（自動判定結果がデフォルト選択）
- 使ったらこうなるはず: 既存ファイル名を入力して確定すると、ダイアログにエラーが表示されて作成が止まる
- 使ったらこうなるはず: ファイル作成後にサイドバーが自動更新される（リロード不要）
- 使ったらこうなるはず: テキストを入力してしばらく止めると、save ボタンを押さなくてもファイルが保存されている
- 使ったらこうなるはず: preview ボタンが存在せず、テキスト入力に応じてプレビューが自動更新される
- 使ったらこうなるはず: ファイル選択セレクトボックスが存在せず、サイドバーからファイルを開ける
- こうなったら外れ: 新規作成したファイルが「公開済み」になる
- こうなったら外れ: テキスト入力後に手動で save を押さないと保存されない
- こうなったら外れ: 保存時にエディタから別のページへリダイレクトされる
- こうなったら外れ: 既存ファイルと同じ名前で新規作成が成功してしまう
