# Findings: editor-ui-cleanup

**日時:** 2026-06-28
**対象:** plans/editor-ui-cleanup/

---

## 操作の事実

1. 「新規作成」ボタン → 画面中央にポップアップダイアログが表示された
2. ダイアログの入力フォームで Enter キーを押す → ダイアログが閉じ、新規ファイルが作成されなかった（入力の有無にかかわらず同じ挙動）
3. 既存ファイルと同名を入力して「作成」→ ダイアログが閉じ、エディタが「新規ファイルっぽい」状態になる。リロードすると既存ファイルが読み込まれる（上書きされていない、エラーも表示されない）
4. テンプレート選択 UI はなく、ファイルパスによる自動判定が行われた
5. ファイル作成後、サイドバーに新規ファイルが表示されなかった。リロードすると表示された
6. テキスト入力後に停止 → プレビューの自動更新と自動保存が両方発生した
7. サイドバーからファイルを選んで開く操作に支障なし

## 発見（解釈・原因の仮説、戻し先つき）

### F-01: Enter キーがキャンセル扱いになっている

**戻し先:** /tdd-run（同一プラン）
**出どころ:** 利用インタビュー
**解釈・原因の仮説:** ダイアログの `<form>` または `<input>` の `keydown` ハンドラで Enter がデフォルト動作（フォームサブミット → ページリロード？）の防止のみ行われ、「作成」相当の処理が呼ばれていない可能性。
**推奨される対応:** 入力フォームで Enter を押したとき、入力値があれば「作成」ボタンと同じ処理を呼ぶ。

---

### F-02: 既存ファイル名での作成がサイレントに失敗する

**戻し先:** /tdd-run（同一プラン）
**出どころ:** 利用インタビュー
**解釈・原因の仮説:** 保存エンドポイント（`/save`）が既存ファイルの有無をチェックしていないか、エラーレスポンスを返しても UI 側でハンドリングしていない。ダイアログが閉じて「新規ファイル」状態になるのは、保存成功・失敗にかかわらず同じ処理をしているため。
**推奨される対応:** 新規作成時はファイルの存在確認を行い、既存の場合はダイアログにエラーを表示して作成を止める。

---

### F-03: テンプレートを選択できない（自動判定のみ）

**戻し先:** /tdd-run（同一プラン）
**出どころ:** 利用インタビュー
**解釈・原因の仮説:** problem.md「新規作成時にファイル名とテンプレートを指定できる」に対し、実装はパスによる自動判定のみ。自動判定が外れたケースはなかったが、「任意のファイルに任意のテンプレートを適用する」という自由度がない。
**推奨される対応:** ダイアログにテンプレート選択セレクトボックスを追加する。自動判定の結果をデフォルト選択として表示しつつ、上書き選択できる形が自然。

---

### F-04: 新規作成後にサイドバーが更新されない

**戻し先:** /tdd-run（同一プラン）
**出どころ:** 利用インタビュー
**解釈・原因の仮説:** 「作成」完了後にサイドバーのツリーを再取得・再レンダリングするトリガーがない。
**推奨される対応:** 新規ファイル作成成功後にサイドバーツリーを再描画する。

---

### F-05: サイドバーに公開ステータスが表示されない（新規問題）

**戻し先:** 新規プラン（/tdd-problem）
**出どころ:** 利用インタビュー（別件として言及）
**解釈・原因の仮説:** 現在のサイドバーはファイル名のみを表示し、公開ステータス（未公開・更新あり・公開済み）を反映していない。複数ファイルを行き来すると、どのファイルが未公開か・更新中かを追跡できなくなる。
**推奨される対応:** 別プランとして「サイドバーへの公開ステータス表示」を立てる。

---

### F-06: ダイアログの枠が太い（軽微）

**戻し先:** /tdd-run（同一プラン）
**出どころ:** 利用インタビュー
**解釈・原因の仮説:** ダイアログの border または box-shadow が現在のデザインに対して目立ちすぎている。
**推奨される対応:** ダイアログの縁を細くする（CSS 調整）。機能ではなくビジュアルの問題。

---

### F-07: 上部 UI のレイアウト整理

**戻し先:** /tdd-run（同一プラン）
**出どころ:** 利用インタビュー
**解釈・原因の仮説:** 現在の上部 UI は要素順・マージン・不要要素の3点で問題がある。ファイルパスは常に存在する優先度の高い情報なのに左端になく、ボタンと隣接テキストに gap がなく、不要な「戻る」リンクが残っている。
**推奨される対応:** 以下のレイアウトに変更する。

```
| ファイル名 | 更新状態(未公開|更新あり|公開済み) | [新規作成] |  （右寄せ）  | [publish] |
```

- `publish` のみ右寄せ（`justify-content: space-between` 等）
- 要素間に適切な gap を付与
- バー下部に 5px のマージンを付与
- 「戻る」リンクは削除（サイドバーがナビゲーションを代替しているため不要）
