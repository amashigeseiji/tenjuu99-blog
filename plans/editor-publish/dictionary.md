**アプリケーションドメイン**

### 公開ステータス

en: PublicationStatus
**定義:** ファイルごとのリモート反映状態を表す状態機械。以下の3ステートをとる。

- **未公開**: ローカルに存在するがリモートに存在しない（新規作成後、または非公開にした後）
- **更新あり**: リモートに存在するがローカルの内容が異なる（公開後に編集した状態）
- **公開済み**: ローカルとリモートの内容が一致している

**遷移アクション（スコープ内）:**
```
◯（未存在）  ──[新規作成する]──→ 未公開
未公開       ──[公開する]    ──→ 公開済み
公開済み     ──[編集する]    ──→ 更新あり
更新あり     ──[更新する]    ──→ 公開済み
未公開/更新あり ──[編集する] ──→ 自己遷移
```

**遷移アクション（スコープ外・将来の拡張）:**
```
公開済み/更新あり ──[非公開にする]──→ 未公開
任意             ──[削除する]    ──→ ◯（未存在）
```

**関係:**
- `references`: [公開する](#公開する) — 未公開 → 公開済み の遷移アクション
- `references`: [更新する](#更新する) — 更新あり → 公開済み の遷移アクション
- `references`: [非公開にする](#非公開にする) — 公開済み/更新あり → 未公開 の遷移アクション（将来実装）
- `references`: [削除する](#削除する) — 任意 → ◯ の遷移アクション（将来実装）
- `references`: [公開ステータス判定器](#公開ステータス判定器) — 現在のステートを導出する
**src:** （ウォークスルーで記入）
**発見:** tdd-run (editor-publish)
**状態:** 再定義

### 公開する

en: Publish
**定義:** [公開ステータス](#公開ステータス)の `未公開 → 公開済み` 遷移を起動するアクション。Publishボタンで実行される。[保存する](#保存)とは別操作（保存はローカル書き込みのみ）。コミットメッセージの入力なしに完結する。
**関係:**
- `references`: [公開ステータス](#公開ステータス) — 遷移を起動する
- `references`: [公開対象](#公開対象) — 自動収集した公開対象セットを対象とする
- `references`: [公開フィードバック](#公開フィードバック) — 操作結果を通知する
- `references`: [デプロイ](#デプロイ) — 公開操作によって起動される
**src:** `packages/editor/server/changeReflector.js`
**発見:** tdd-run (editor-publish)
**状態:** 新規

### 更新する

en: Update
**定義:** [公開ステータス](#公開ステータス)の `更新あり → 公開済み` 遷移を起動するアクション。公開済みのファイルに編集が加わった後に実行される。[公開する](#公開する)と同じボタンで実行されるが、意味的に異なる（既存コンテンツの改訂）。
**関係:**
- `references`: [公開ステータス](#公開ステータス) — 遷移を起動する
- `references`: [公開対象](#公開対象) — 自動収集した公開対象セットを対象とする
- `references`: [公開フィードバック](#公開フィードバック) — 操作結果を通知する
**src:** `packages/editor/server/changeReflector.js`
**発見:** tdd-run (editor-publish)
**状態:** 新規

### 公開済み状態

en: PublishedState
**定義:** リモートリポジトリの現在の内容を参照するための読み取り専用の抽象。この抽象を介して[公開ステータス判定器](#公開ステータス判定器)がローカルとリモートを対比し、現在のステートを導出する。
**実装例:** gitの場合はremote HEADへのアクセス（`existsInRemote`, `diffFromRemote`）にあたる。
**関係:**
- `references`: [公開ステータス判定器](#公開ステータス判定器) — に注入されて使われる
**src:** （ウォークスルーで記入）
**発見:** tdd-run (editor-publish)
**状態:** 再定義（書き込み操作を分離、読み取り専用に限定）

### 公開ハンドラー

en: PublishHandler
**定義:** HTTPリクエストを受け取り、[公開対象コレクター](#公開対象コレクター)で対象を収集し、[変更反映器](#変更反映器)で[公開する](#公開する)または[更新する](#更新する)遷移を実行し、[公開フィードバック](#公開フィードバック)をレスポンスとして返す装置。ファイルの保存も一括して処理する（F-04対応）。

### 公開ステータス判定器

en: PublicationStatusResolver
**定義:** [公開済み状態](#公開済み状態)を注入されて、ファイルパスから現在の[公開ステータス](#公開ステータス)（未公開/更新あり/公開済み）を導出する装置。純粋関数として実装され、注入された抽象を介すことでgitなしにテストできる。
**関係:**
- `references`: [公開済み状態](#公開済み状態) — 注入されるリモート参照抽象
- `references`: [公開ステータス](#公開ステータス) — 導出して返す
**src:** `packages/editor/server/publicationStatus.js`

### 公開対象コレクター

en: PublishTargetCollector
**定義:** 現在のファイルパスと本文から[公開対象](#公開対象)を収集する装置。[画像参照抽出器](#画像参照抽出器)を使って参照画像パスを取得し、Markdownファイルと合わせたファイルリストを返す。
**関係:**
- `references`: [公開対象](#公開対象) — 収集した結果が公開対象にあたる

**定義:** Markdown文字列から `![](...)` 形式の画像パスを抽出する装置。純粋関数として実装され、ファイルシステムへの依存を持たない。
**関係:**
- `references`: [公開対象コレクター](#公開対象コレクター) — に組み込まれる
**src:** `packages/editor/js/imageReferenceExtractor.js`

### 変更反映器

en: ChangeReflector
**定義:** [公開する](#公開する)または[更新する](#更新する)の遷移イベントを受けて、[公開対象](#公開対象)のファイル群をリモートに反映する装置。遷移の実行体（commit/push）を外部から注入できることで、実際のリモートリポジトリなしにテストが書ける。
**関係:**
- `references`: [公開対象](#公開対象) — 反映する対象を受け取る
**src:** `packages/editor/server/changeReflector.js`
