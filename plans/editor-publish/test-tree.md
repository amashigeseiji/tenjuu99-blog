# テストツリー: editor-publish

**作成:** 2026-06-27
**更新:** 2026-06-27 (v2 - 状態機械による再設計)

## できるのツリー

```
エディタは編集内容を公開・更新できる [editor]
├── 公開ボタンはリクエストを送信してフィードバックを表示できる [editor]  ← ブラウザAPI依存: 手動確認のみ
└── 公開ハンドラーは保存と遷移を一括で処理できる [editor]
    ├── 公開対象コレクターは現ファイルと参照画像を収集できる [editor]
    │   └── 画像参照抽出器はMarkdownから画像パスを抽出できる [editor]
    ├── 変更反映器は遷移実行体を介してリモートに反映できる [editor]
    └── 公開ステータス判定器は公開済み状態からステータスを導出できる [editor]
```

## 利用仮説

- 使ったらこうなるはず: Publishボタンを押すとターミナル操作なしにGitHubへのpushが実行され、編集画面に結果（完了または失敗）が表示される
- 使ったらこうなるはず: push対象には現在編集中のファイルと、そのMarkdownから参照されるすべての画像が含まれる
- 使ったらこうなるはず: 公開ステータス（未公開/更新あり/公開済み）がファイルごとに表示される
- こうなったら外れ: Publishボタンを押したのにターミナルで手動操作が必要になる
- こうなったら外れ: 保存操作がサーバーリロードを引き起こし、直後のpublishリクエストが失敗する
- こうなったら外れ: 失敗時に「Failed to fetch」のような技術的なエラー文字列がそのまま表示される
