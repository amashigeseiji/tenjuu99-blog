# テストツリー: publish-frontmatter-image

**作成:** 2026-07-15

## できるのツリー

```
公開対象コレクターは記事由来のすべての画像参照から公開対象を収集できる [editor]
├── 画像参照抽出器は本文から画像参照を抽出できる [editor]
│   ├── 画像参照抽出器は Markdown 画像記法の参照を抽出できる [editor] （既存挙動）
│   └── 画像参照抽出器は本文に埋め込まれた img タグの参照を抽出できる [editor] （新規）
└── フロントマター画像参照抽出器は frontmatter から画像参照を抽出できる [editor]
    ├── フロントマター解析器は記事の文字列から frontmatter の値を取り出せる [ssg-core] （既存実装の再利用）
    └── 画像パス判定器は値が画像パスの形かどうか判定できる [editor]
```

**エントリーポイント:** `packages/editor/server/publish.js`（公開ハンドラー） → 公開対象コレクター

**設計メモ:**
- フロントマター解析器は `lib/pageData.js` 内の既存実装（`parseMetaData`）を named export に昇格して再利用する（独自パーサーの複製を避ける）
- 画像パス判定は値の形（ローカルパス + 画像拡張子）で行う。フィールド名では判別しない
- パターン適用なし（2系統の抽出結果を合流させる集約構造）

## 利用仮説

- 使ったらこうなるはず: `og_image` に画像パスを指定した記事を公開すると、その画像がリモートに送られ、公開ページで OGP 画像が表示される
- 使ったらこうなるはず: 本文（記法・img タグ）と frontmatter の画像が両方とも公開対象に含まれる
- 使ったらこうなるはず: 本文参照のみの記事は従来どおりの公開対象になる
- こうなったら外れ: タイトルやテンプレート名など画像でない frontmatter 値がファイルとして公開対象に混ざる
- こうなったら外れ: frontmatter に画像を書いても公開対象が Markdown ファイルと本文画像だけになる
