## US-01: コンテンツルートの状態によらず同梱機能が動作する

**ロール:** 配布物（`.app`）の利用者
**目的:** コンテンツルートの状態（実体 `node_modules` の有無や中身）に関わらず、追加インストールなしに同梱されている機能（画像メタデータ読み取り等）を使いたい
**価値:** 配布先環境で追加のセットアップ作業を強いられずに、配布物単体で機能一式が動作する

### シナリオ 1: 実体 node_modules を持たないコンテンツルートでの動作

```
Given 実体の node_modules を持たない（追加インストールを一切していない）フォルダをコンテンツルートに選ぶ
When 配布物（`.app`）を起動し、画像メタデータ読み取り機能を使う
Then コンテンツルート側に何もインストールすることなく、機能が正常に動作する
```

### シナリオ 2: 実体 node_modules を持つ開発中プロジェクトとの同居

```
Given 実体の node_modules を持つ開発中プロジェクトをコンテンツルートに選ぶ
When 配布物（`.app`）を起動し、画像メタデータ読み取り機能を使う
Then コンテンツルート側の既存ファイル・ディレクトリが一切変更・削除されないまま、機能が正常に動作する
```
