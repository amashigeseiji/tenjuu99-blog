## US-01: ブラウザの編集画面から変更を公開する

**ロール:** 非技術者（コンテンツ編集者）
**目的:** 編集した内容をターミナルを操作せずにWebサイトに公開したい
**価値:** デプロイのたびに技術者に依頼する必要がなくなり、コンテンツ公開を自分で完結できる

### シナリオ 1: Publishボタンを押すと変更がGitHubにpushされる

```
Given 編集した内容がローカルに保存されている
When 編集画面のPublishボタンを押す
Then ターミナルを一切操作せずにGitHubへのpushが実行される
And コミットメッセージの入力は不要で操作が完結する
```

### シナリオ 2: 公開完了がブラウザで確認できる

```
Given Publishボタンを押してpush操作が実行された
When pushが正常に完了する
Then 公開が完了したことを示すフィードバックが編集画面に表示される
```

### シナリオ 3: 公開失敗がブラウザで確認できる

```
Given Publishボタンを押してpush操作が実行された
When git操作が何らかの理由で失敗する
Then 失敗したことを示すフィードバックが編集画面に表示される
And ターミナルを操作せずに失敗を知ることができる
```
