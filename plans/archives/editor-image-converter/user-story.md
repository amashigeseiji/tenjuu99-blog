## US-01: アップロード時の画像変換

**ロール:** ブログ執筆者
**目的:** 画像をアップロードするだけで、Webに適したサイズ・フォーマットで保存されてほしい
**価値:** アップロード前に手動で画像を変換する作業が不要になる

### シナリオ 1: 画像をアップロードすると変換されて保存される

```
Given blog.json に image_converter として "sharp" が設定されている
When 編集画面から画像ファイルをアップロードする
Then 元のサイズ・フォーマットではなく、Webに適したサイズ・フォーマットに変換された画像が保存される
```

### シナリオ 2: sharp がインストール済みなので追加作業なく変換が動作する

```
Given @tenjuu99/blog をインストールした環境である
When blog.json に image_converter として "sharp" を設定する
Then ユーザーが別途 sharp をインストールしなくても画像変換が動作する
```

### シナリオ 3: 変換設定をしていない場合は従来どおり動作する

```
Given blog.json に image_converter が設定されていない
When 編集画面から画像ファイルをアップロードする
Then 元のファイルがそのまま保存される（変換なし・従来と同じ挙動）
```

## US-02: ビルド時の画像変換

**ロール:** ブログ運営者
**目的:** generate を実行するだけで、ソースの画像がWebに適した状態で dist に出力されてほしい
**価値:** ビルド前の手動変換作業が不要になる

### シナリオ 1: generate するとソース画像が変換されて出力される

```
Given blog.json に画像変換の設定がされている
And src/ ディレクトリに画像ファイルが存在する
When generate を実行する
Then dist/ に出力された画像が、Webに適したサイズ・フォーマットに変換されている
```

### シナリオ 2: 変換設定がない場合は従来どおりコピーされる

```
Given blog.json に画像変換の設定がされていない
And src/ ディレクトリに画像ファイルが存在する
When generate を実行する
Then dist/ に画像がそのままコピーされる（変換なし・従来と同じ挙動）
```
