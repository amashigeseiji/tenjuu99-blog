# テストツリー: error-visibility

**作成:** 2026-07-08

**エントリーポイント:** native/Sources/App/AppDelegate.swift → ネイティブアプリシェル

## できるのツリー

```
ネイティブアプリシェルは 正常に使えない状態になったとき何が起きたかを利用者に知らせることができる [native-shell]
├── サーバーライフサイクル連動は サーバープロセスの出力と予期しない終了を観測できる [native-shell]
├── 診断ログ書き出しは サーバー出力を直近数回の起動分だけ残せる [native-shell]
├── 表示状態判定器は サーバーの生死と起動完了の観測から失敗段階を含む表示状態を判定できる [native-shell]
└── エラー表示は 失敗段階とエラー内容と診断ログの場所から表示内容を組み立てられる [native-shell]
```

- テスト置き場: `native/Tests/NativeShellCoreTests/`（自作ハーネス、`swift run CoreTests`）
- 画面への提示そのもの（AppKit 配線）は AppDelegate で行い、既存プラン同様に手動検証。
  単体テスト対象は表示内容の組み立てまで。

## 利用仮説

- 使ったらこうなるはず:
  - サーバーが起動に失敗すると、真っ白ではなく「起動に失敗した」こととエラー内容が画面に出る
  - 起動後にサーバーが落ちると、「起動後に問題が起きた」ことが起動失敗と区別されて画面に出る
  - 正常時にはエラーに関する表示は一切現れない
  - 診断ログは直近数回の起動分だけ残り、無限に溜まらない
- こうなったら外れ:
  - 異常時に真っ白な画面だけが残る
  - 失敗がどの段階で起きたか画面から区別できない
  - ログファイルが起動のたびに増え続ける
