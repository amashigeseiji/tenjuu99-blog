## US-01: ノンエンジニアがNode未インストールのMacでアプリを使う

**ロール:** Node.jsが入っていないMacを使うノンエンジニアの配布先ユーザー
**目的:** 追加のインストール作業なしに、`.app` をダブルクリックするだけで `@tenjuu99/blog` を使いたい
**価値:** Node/npm等の専門知識がなくても、渡された配布物をそのまま使い始められる

### シナリオ 1: 正常系 - 初回起動から編集操作まで

```
Given Node.jsやsharpがインストールされていないMacに、配布された `.app` が置かれている
When ユーザーが `.app` をFinderからダブルクリックする
Then ネイティブアプリのウィンドウが開き、`@tenjuu99/blog` の起動〜編集操作（保存・ツリー操作・プレビュー等）が一通り行える
```

### シナリオ 2: 境界系 - 未署名アプリのGatekeeper警告

```
Given 配布された `.app` が署名されていない
When ユーザーが初めて `.app` をダブルクリックする
Then macOSが「開発元が未確認のため開けません」という警告を表示するが、ユーザーが右クリック→「開く」を選ぶことで起動を継続できる
```

## US-02: 配布物が外部依存を必要とせず自己完結している

**ロール:** 配布物を作成する開発者（自分）
**目的:** `.app` 単体でNode本体・sharp・ソースコード一式が完結しており、配布先の環境に依存しないことを確認したい
**価値:** 配布後に「動かない」という問い合わせが発生しない

### シナリオ 1: 正常系 - クリーンな環境での自己完結性

```
Given `.app` がNode未インストール・sharp未インストールのクリーンなMac環境に置かれている
When `.app` を起動する
Then Node本体・sharpのネイティブバイナリ・ソースコード（`lib/`, `packages/` 等）が `.app` 内部から解決され、外部の `node` コマンドやグローバルインストール済みパッケージに依存せず動作する
```
