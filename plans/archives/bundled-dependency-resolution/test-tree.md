# テストツリー: bundled-dependency-resolution

**作成:** 2026-07-17

## できるのツリー

同梱モジュール解決器は 同梱コードが使う依存パッケージへの参照を、コンテンツルートの中身に関わらず同梱コード自身の実体へ解決できる [app-bundle]
├── 同梱モジュール解決器は アプリ自身のパッケージ名への参照を同梱コードへ解決できる [app-bundle]  （既存・実装済み、回帰確認のみ）
├── 依存パッケージ名判定器は 参照が同梱コード自身の依存パッケージへの参照かどうか判定できる [app-bundle]
│   └── 依存パッケージ名判定器は 同梱コード自身が持つ依存パッケージ名の一覧を取得できる [app-bundle]
└── 同梱モジュール解決器は 依存パッケージ名判定器が一致と判定した参照を同梱コード自身の実体へ解決させられる [app-bundle]

## 利用仮説

- 使ったらこうなるはず: 実体の node_modules を持たないコンテンツルートで配布物（`.app`）を起動し、画像メタデータ読み取り機能を使うと、コンテンツルート側に何もインストールすることなく機能が正常に動作する（US-01 シナリオ1）
- 使ったらこうなるはず: 実体の node_modules を持つ開発中プロジェクトをコンテンツルートに選んで配布物を起動しても、コンテンツルート側の既存ファイル・ディレクトリが一切変更・削除されないまま機能が正常に動作する（US-01 シナリオ2）
- こうなったら外れ: `Cannot find package '<依存パッケージ名>'` のようなエラーで起動時にサーバーがクラッシュする（依存パッケージ名判定に漏れがある、または解決委譲が機能していない）
- こうなったら外れ: コンテンツルート側に `node_modules` への書き込みやシンボリックリンクが発生する（既存物保護に反する手段で解決している）
