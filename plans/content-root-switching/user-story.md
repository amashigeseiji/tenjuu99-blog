## US-01: 別のプロジェクトへ切り替える

**ロール:** 同一 Mac 内に複数のプロジェクト（それぞれ `blog.json` を持つコンテンツフォルダ）を持つ利用者
**目的:** 単一のネイティブアプリで、複数のプロジェクトをそれぞれ使いたい
**価値:** プロジェクトごとに特別な操作や環境の作り直しをせず、同じアプリで行き来できる

### シナリオ 1: アプリの操作でコンテンツフォルダを変更する

```
Given アプリがプロジェクト A をコンテンツルートとして起動している
And   同じ Mac 内に blog.json を持つ別のプロジェクト B がある
When  利用者がアプリの通常の操作の範囲でコンテンツフォルダの変更を選び、プロジェクト B を指定する
Then  アプリはプロジェクト B の内容で使える状態になる
And   永続化ストアの手動操作など、アプリ外部の知識を必要としない
```

### シナリオ 2: 切り替え後の記憶が引き継がれる

```
Given 利用者がプロジェクト A からプロジェクト B へ切り替えた
When  アプリを終了して再起動する
Then  ダイアログなしでプロジェクト B のまま立ち上がる
```

### シナリオ 3: 切り替え操作を途中でやめる

```
Given アプリがプロジェクト A をコンテンツルートとして起動している
When  利用者がコンテンツフォルダの変更操作を始めたが、フォルダを選択せずにやめる
Then  アプリはプロジェクト A のまま使い続けられる
```
